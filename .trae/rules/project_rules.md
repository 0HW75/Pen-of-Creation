# 项目开发规则

## 数据库字段修改同步规范

**核心原则：修改数据库字段时，必须同时修改前端和后端代码。**

### 修改字段时的必做检查项

1. **后端修改**
   - 数据库模型（Model）
   - API 接口（接收参数、返回数据）
   - 数据库迁移脚本（如有）

2. **前端修改**
   - 表单字段名
   - 表单验证规则
   - 列表展示列
   - API 调用参数

### 字段映射规则

前后端字段名不一致时，在 `handleSubmit` 中转换：

```javascript
const data = {
  name: values.xxx_name,  // 前端 -> 后端
  world_id: worldId,
  // ...
};
await api.create(data);
```

### 禁止行为

- ❌ 只改前端不改后端
- ❌ 只改后端不改前端
- ❌ 前后端字段不一致且不转换

### 修改流程

修改字段前 → 检查前端使用位置 → 检查后端使用位置 → 同时修改 → 测试创建/编辑/列表功能
