## 问题分析

通过对比前端模块配置和后端模型实现，我发现了关键差异：

### 前端期望
所有设定类型（角色、地点、物品、势力等）都有 `name` 和 `description` 属性，前端表格显示这两个字段。

### 后端实现
- **有 description 属性的模型**：CharacterTrait、CharacterAbility、CharacterRelationship、FactionStructure、FactionGoal、LocationStructure、SpecialLocation、EquipmentSystem、SpecialItem、Relationship
- **缺少 description 属性的模型**：Character、Location、Item、Faction、WorldSetting、EnergySystem、SocietyCulture、History、Ability、Skill、Talent、Race、Creature、SpecialCreature、Timeline、DataAssociation

## 根本原因

- 当用户提交表单时，前端发送了包含 `description` 字段的数据
- 后端的许多模型没有 `description` 属性，所以忽略了这个字段
- 当后端返回数据时，没有 `description` 字段，前端的表格无法正确显示
- 对于有 `description` 属性的模型（如 CharacterTrait），可以正常显示

## 解决方案

为所有缺少 `description` 属性的模型添加该属性，使其与前端期望的结构一致。

## 实现步骤

1. **修改 `backend/app/models.py` 文件**：
   - 为 `Character` 模型添加 `description` 字段
   - 为 `Location` 模型添加 `description` 字段
   - 为 `Item` 模型添加 `description` 字段
   - 为 `Faction` 模型添加 `description` 字段
   - 为 `WorldSetting` 模型添加 `description` 字段
   - 为 `EnergySystem` 模型添加 `description` 字段
   - 为 `SocietyCulture` 模型添加 `description` 字段
   - 为 `History` 模型添加 `description` 字段
   - 为 `Ability` 模型添加 `description` 字段
   - 为 `Skill` 模型添加 `description` 字段
   - 为 `Talent` 模型添加 `description` 字段
   - 为 `Race` 模型添加 `description` 字段
   - 为 `Creature` 模型添加 `description` 字段
   - 为 `SpecialCreature` 模型添加 `description` 字段
   - 为 `Timeline` 模型添加 `description` 字段
   - 为 `DataAssociation` 模型添加 `description` 字段

2. **修改对应模型的 `to_dict()` 方法**：
   - 在各模型的 `to_dict()` 方法中添加 `description` 字段

3. **修改对应 API 的 `create_*` 函数**：
   - 在各模型的 `create_*` 函数中添加对 `description` 字段的处理

4. **修改对应 API 的 `update_*` 函数**：
   - 在各模型的 `update_*` 函数中添加对 `description` 字段的处理

## 预期效果

- 所有设定类型都能在前端正确显示
- 创建设定时，`description` 字段能够被正确保存
- 编辑设定时，`description` 字段能够被正确更新
- 所有设定类型的显示效果保持一致
